# Seedance 2.0 计费规则说明

本文说明当前 `newapi` 中 `doubao-seedance-2-0-260128`、`doubao-seedance-2-0-fast-260128` 的计费规则、后台配置方式，以及 `newapi` 如何根据分辨率和输入内容自动换算最终价格。

## 适用模型

- `doubao-seedance-2-0-260128`
- `doubao-seedance-2-0-fast-260128`

## 一、火山官方价格

### 1. `doubao-seedance-2-0-260128`

#### 输出分辨率为 `480p / 720p`

- 输入不含视频：`46.00`
- 输入包含视频：`28.00`

#### 输出分辨率为 `1080p`

- 输入不含视频：`51.00`
- 输入包含视频：`31.00`

### 2. `doubao-seedance-2-0-fast-260128`

#### 输出分辨率为 `480p / 720p`

- 输入不含视频：`37.00`
- 输入包含视频：`22.00`

#### 输出分辨率为 `1080p`

- 当前不支持单独的 `1080p` 价格规则
- 在现有代码中按“无 1080p 价格表”处理，不会追加 `resolution` 倍率


## 二、`newapi` 当前实现规则

当前实现约定：

1. 后台模型基础价格应配置为：
   `480p/720p 且输入不含视频` 的官方价格

2. `newapi` 在请求进入时自动检查：
   - 是否包含 `video_url` 输入
   - 是否传了 `resolution=1080p`

3. 如果命中这些条件，会自动追加 `OtherRatios`：
   - `video_input`
   - `resolution`

4. 最终任务预扣费/倍率基数按：

```text
基础价格 × video_input倍率 × resolution倍率
```

## 三、后台应该怎么配置

### 1. `doubao-seedance-2-0-260128`

后台模型基础价格应配置为：

- `46.0 / 1M tokens`

也就是“不含视频 + 480p/720p”的官方价格。

### 2. `doubao-seedance-2-0-fast-260128`

后台模型基础价格应配置为：

- `37.0 / 1M tokens`

也就是“不含视频 + 480p/720p”的官方价格。

## 四、系统如何自动换算

### 1. 视频输入折扣倍率

#### `doubao-seedance-2-0-260128`

```text
video_input = 28 / 46 ≈ 0.6086956522
```

#### `doubao-seedance-2-0-fast-260128`

```text
video_input = 22 / 37 ≈ 0.5945945946
```

### 2. 1080P 分辨率倍率

#### `doubao-seedance-2-0-260128`

不含视频时：

```text
resolution = 51 / 46 ≈ 1.1086956522
```

含视频时：

```text
resolution = 31 / 28 ≈ 1.1071428571
```

#### `doubao-seedance-2-0-fast-260128`

- 当前没有 1080P 价格表
- 不追加 `resolution` 倍率

## 五、四种典型场景的最终价格

### `doubao-seedance-2-0-260128`

#### 场景 1：480p/720p，不含视频

```text
基础价格 = 46
最终价格 = 46
```

#### 场景 2：480p/720p，含视频

```text
基础价格 = 46
video_input = 28 / 46
最终价格 = 46 × (28 / 46) = 28
```

#### 场景 3：1080p，不含视频

```text
基础价格 = 46
resolution = 51 / 46
最终价格 = 46 × (51 / 46) = 51
```

#### 场景 4：1080p，含视频

```text
基础价格 = 46
video_input = 28 / 46
resolution = 31 / 28
最终价格 = 46 × (28 / 46) × (31 / 28) = 31
```

### `doubao-seedance-2-0-fast-260128`

#### 场景 1：480p/720p，不含视频

```text
基础价格 = 37
最终价格 = 37
```

#### 场景 2：480p/720p，含视频

```text
基础价格 = 37
video_input = 22 / 37
最终价格 = 37 × (22 / 37) = 22
```
