# 分组与渠道折扣设置教程

本教程介绍如何通过"分组"机制实现多客户差异化定价。例如为 VIP 客户提供 5 折优惠，为 SVIP 客户提供 3 折优惠。

---

## 一、核心概念

### 1.1 什么是分组

分组是一个定价维度，用于控制不同客户群体使用 API 时的费率。

```
最终扣费 = 模型价格 × 分组倍率
```

| 分组 | 倍率 | 含义 |
|------|------|------|
| default | 1.0 | 原价 |
| vip | 0.5 | 5 折 |
| svip | 0.3 | 3 折 |

### 1.2 分组的关联关系

```
用户(User) ──→ 所属分组(Group)
    │
    └──→ 创建令牌(Token) ──→ 令牌分组(可选择任何可用分组)
                │
                └──→ 请求匹配渠道(Channel) ──→ 渠道必须包含该分组
```

**关键链路：**
1. 管理员给用户设置分组（如 vip）
2. 用户创建令牌时选择分组
3. 请求到达时，系统根据令牌分组匹配已配置该分组的渠道
4. 扣费时按分组倍率计算

---

## 二、完整配置步骤

### 步骤 1：创建分组并设置倍率

**路径：** 系统设置 → 运营设置 → 分组与模型定价设置 → 分组设置

在可视化编辑模式下的「分组管理」表格中操作：

| 操作 | 说明 |
|------|------|
| 点击「添加分组」 | 新增一行 |
| 分组名称 | 填写英文标识，如 `vip`、`svip`（不可包含中文或空格） |
| 分组倍率 | 折扣系数。`1` = 原价，`0.5` = 5 折，`0.3` = 3 折，`0` = 免费 |
| 用户可选 | 勾选后，用户创建令牌时可以选择此分组 |
| 描述 | 显示给用户的名称，如 "VIP分组"、"SVIP分组" |

**示例配置：**

| 分组名称 | 倍率 | 用户可选 | 描述 |
|---------|------|:--------:|------|
| default | 1.0 | ✅ | 默认分组 |
| vip | 0.5 | ✅ | VIP分组 |
| svip | 0.3 | ✅ | SVIP分组 |

对应的 JSON 配置（手动编辑模式）：

**分组倍率 (GroupRatio)：**
```json
{
  "default": 1,
  "vip": 0.5,
  "svip": 0.3
}
```

**用户可选分组 (UserUsableGroups)：**
```json
{
  "default": "默认分组",
  "vip": "VIP分组",
  "svip": "SVIP分组"
}
```

点击「保存分组设置」。

---

### 步骤 2：为渠道分配分组

**路径：** 渠道管理 → 编辑渠道

每个渠道都有一个「分组」字段，用于指定该渠道对哪些分组可用。

**场景 A：所有分组共用同一批渠道**

编辑渠道，在分组字段中选择所有需要的分组：

```
分组: default, vip, svip
```

这样无论用户是哪个分组，请求都会路由到此渠道，但扣费按各自分组倍率计算。

**场景 B：不同分组使用不同渠道（推荐）**

创建多个渠道指向同一个上游，但分配不同的分组：

| 渠道名称 | 上游地址 | 分组 | 优先级 | 备注 |
|---------|---------|------|:------:|------|
| OpenAI-默认 | api.openai.com | default | 0 | 普通用户 |
| OpenAI-VIP | api.openai.com | vip | 0 | VIP 专属，可配置更高并发 |
| OpenAI-SVIP | api.openai.com | svip | 0 | SVIP 专属 |

好处：
- 可为高价值客户分配更快/更稳定的渠道
- 某个分组的渠道出问题不影响其他分组
- 可单独设置每个分组渠道的权重、限速

---

### 步骤 3：给用户分配分组

**路径：** 用户管理 → 编辑用户

找到目标用户，修改「分组」字段：

| 用户 | 分组 |
|------|------|
| 张三（普通客户） | default |
| 李四（VIP客户） | vip |
| 王五（SVIP客户） | svip |

保存后立即生效（有缓存延迟约几秒）。

---

### 步骤 4：用户创建令牌时选择分组

用户登录后，在「令牌管理」中创建新令牌时，会看到一个分组下拉框。

- **default 分组用户**：只能选择 default 分组
- **vip 分组用户**：可以选择 default 和 vip（取决于「用户可选分组」配置）
- **svip 分组用户**：可以选择 default、vip 和 svip

> 用户所属分组决定了 ta 能看到哪些分组选项。如果想让 vip 用户也能选择 svip 分组，需要配置「特殊可用分组规则」（见高级配置）。

---

## 三、计费示例

假设模型 `gpt-4o` 定价 = 60（每百万 token 输入价格），用户发起一次请求消耗 1000 token：

| 用户分组 | 分组倍率 | 计算 | 实际扣费 |
|---------|---------|------|---------|
| default | 1.0 | 60 × 1.0 | 60 |
| vip | 0.5 | 60 × 0.5 | 30 |
| svip | 0.3 | 60 × 0.3 | 18 |

## 四、高级配置

### 4.1 分组间特殊倍率 (GroupGroupRatio)

场景：VIP 用户使用 default 分组的令牌时，希望也享受 0.9 折优惠。

**路径：** 分组设置页面 → 「分组间特殊倍率规则」

```json
{
  "vip": {
    "default": 0.9
  },
  "svip": {
    "default": 0.5,
    "vip": 0.4
  }
}
```

含义：
- vip 用户使用 default 分组时，倍率为 0.9（而非 default 的 1.0）
- svip 用户使用 default 分组时，倍率为 0.5；使用 vip 分组时，倍率为 0.4

**优先级：** 分组间特殊倍率 > 分组基础倍率

### 4.2 特殊可用分组规则 (GroupSpecialUsableGroup)

控制某个用户分组能额外看到或被隐藏的分组选项。

```json
{
  "vip": {
    "+:svip_trial": "SVIP体验分组",
    "-:default": ""
  }
}
```

含义：
- vip 用户创建令牌时，额外显示 `svip_trial` 分组
- vip 用户创建令牌时，隐藏 `default` 分组

语法规则：
- `+:分组名`：添加到可选列表
- `-:分组名`：从可选列表移除
- `分组名`（无前缀）：直接添加到可选列表

### 4.3 自动分组 (AutoGroup)

自动分组用于令牌级别的智能路由。当令牌的分组设置为"自动"时，系统按配置的分组优先级依次尝试匹配渠道。

**路径：** 分组设置页面 → 「自动分组」

```json
["default", "vip"]
```

含义：系统先尝试用 default 分组匹配渠道，全部失败后再尝试 vip 分组。

> 自动分组适合高可用场景：用一个令牌覆盖多个分组的渠道池，自动 failover。

## 五、配置检查清单

完成分组配置后，按此清单逐项验证：

- [ ] **分组倍率已设置**：系统设置 → 分组设置 中包含所有分组及正确倍率
- [ ] **用户可选分组已勾选**：需要用户可选的分组已勾选「用户可选」
- [ ] **渠道已分配分组**：每个渠道的分组字段包含对应分组名
- [ ] **渠道已配置模型**：渠道支持的模型列表已正确填写
- [ ] **用户分组已设置**：目标用户的分组字段已修改为正确值
- [ ] **令牌已创建**：用户已用目标分组创建了新令牌
- [ ] **测试验证**：用新令牌发起请求，查看日志确认分组倍率和扣费正确

## 六、常见问题

### Q: 修改分组倍率后，已有的令牌会受影响吗？
A: 会。分组倍率是实时生效的，下一次请求就会使用新的倍率。

### Q: 用户更换分组后，之前创建的令牌怎么办？
A: 已创建的令牌保留原分组不变。用户需要创建新令牌来使用新分组。如果之前令牌选择的分组与渠道不匹配，请求会报错。

### Q: 分组倍率设为 0 是免费吗？
A: 是的。倍率为 0 时扣费为 0，相当于免费使用。

### Q: 一个渠道可以同时属于多个分组吗？
A: 可以。在渠道编辑页面的分组字段中选择多个分组即可。

### Q: 如何实现"VIP 用户能看到更多模型"？
A: 创建仅分配给 vip 分组的渠道，并在该渠道中配置额外模型。这些模型只有 vip 分组的令牌才能访问。
