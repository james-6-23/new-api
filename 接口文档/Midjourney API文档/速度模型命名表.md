# Midjourney 速度模型命名表

所有支持 `mode` 的接口均可通过 `model` 字段指定速度变体，实现按速差异化计费。

## 命名规则

- 常规动作：`mj_{speed}_{action}`，如 `mj_fast_blend`
- SwapFace：`swap_face_{speed}`，如 `swap_face_fast`
- 基础模型名（如 `mj_blend`）继续有效，等价于不指定速度时的默认行为

## 不支持速度变体的动作

| 动作 | 模型名 | 原因 |
|------|--------|------|
| MODAL | mj_modal | 上游接口无 mode 字段 |
| INPAINT | mj_inpaint | 内部动作，不单独计费 |
| CUSTOM_ZOOM | mj_custom_zoom | 内部动作，不单独计费 |

## 使用方式

客户端在请求体中通过 `model` 字段指定速度模型：

```json
{
  "model": "mj_fast_blend",
  "base64Array": ["data:image/png;base64,xxx1", "data:image/png;base64,xxx2"]
}
```

也可以通过传统的 `mode` 字段指定（不传 `model`）：

```json
{
  "mode": "FAST",
  "base64Array": ["data:image/png;base64,xxx1", "data:image/png;base64,xxx2"]
}
```

两种方式等价。当同时传递 `model` 和 `mode` 时，`model` 优先。

## 计费说明

速度变体模型必须在「系统设置 → 分组与模型定价设置」中单独配置价格，未配置价格的速度变体将拒绝请求并返回错误。
